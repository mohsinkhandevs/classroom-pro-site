Privacy Policy

Effective Date: May 21, 2026  ·  Product: Classroom Pro Downloader  ·  Version: v5.1.0

🚫

Zero Analytics or Tracking

No logins, no analytics SDKs, no database sync, and no tracking scripts. We collection absolutely nothing.

💻

100% Local Pipeline

Course scanning, ZIP file generation, and file compilation are done entirely in your browser memory.

This Privacy Policy explains how Classroom Pro Downloader protects your information. By utilizing this browser extension, you consent to the practices detailed below. This policy complies strictly with the Google API Services User Data Policy, including its strict Limited Use requirements.

1 Zero Data Collection Philosophy

Classroom Pro Downloader does NOT collect, store, transmit, log, or share any of your personal details, Google account information, course content, or usage analytics with any external server — including ours.

The extension is completely decentralized. It has no backend servers, no customer databases, and no cloud-hosted APIs. It functions purely as a sandboxed, client-side browser plugin executing directly in your local environment and communicating exclusively with official Google API endpoints on your behalf.

  • No Accounts: No user registration, authentication portals, or third-party sign-ups.
  • No Telemetry: No telemetry suites, crash collectors, or event trackers are packaged with the code.
  • No Exposure: The Developer cannot view, access, or manipulate any classroom data, enrolled courses, or materials scanned.

2 Google API Scopes & Purpose

To identify and organize course files, the extension asks for authorization to access specific Google API scopes during sign-in. We rely strictly on a least-privilege layout, requesting only the exact access required to scan materials:

Google API Scope Type Functional Purpose
classroom.courses.readonly Read-Only Retrieves lists of active courses you are enrolled in to let you select which classes to scan.
classroom.coursework.me.readonly Read-Only Reads coursework lists, assignments, and structural information containing course material attachments.
classroom.coursework.students.readonly Read-Only Accesses student-visible coursework and teacher attachments published in your courses.
classroom.courseworkmaterials.readonly Read-Only Reads static class materials posted directly to the Course Stream or Coursework page.
classroom.announcements.readonly Read-Only Scans course announcements to find and download any files or shortcuts attached within stream updates.

You can view, manage, and instantly revoke the extension's API authorizations at any time by visiting your official Google Account control panel at myaccount.google.com/permissions.

3 Safe downloads with Zero Google Drive Scopes

Traditional tools request highly privileged Google Drive read/write scopes (like drive.readonly or drive) to download materials. These scopes grant full visibility into all your personal files, spreadsheets, and private folders inside Google Drive, presenting an unnecessary security risk.

Zero Google Drive Permissions: Classroom Pro Downloader operates under a strict, modern safety model. It requests **zero Google Drive scopes**.

Instead of requesting Drive API scopes, our background script processes download calls client-side using secure, cookie-authenticated browser requests. The background worker uses the user's active, pre-existing Google Classroom cookies and active session directly (via credentials: 'include' fetches), fetching attachment assets safely in the background. This local direct download model keeps your Google Drive 100% private and protected.

4 Google Limited Use Disclosure

Classroom Pro Downloader's use and transfer of information received from Google APIs adheres strictly to the Google API Services User Data Policy, including the rigid Limited Use requirements:

  • Sole Utility: Google API data is fetched only to provide the utility you explicitly request — scanning coursework and assembling zip folders. It is never used for secondary functions.
  • Zero Advertising: No data obtained through Google APIs is analyzed, repurposed, or transferred to third-party ad networks or data aggregators.
  • No Transference: Your credentials, tokens, and course details are never sold, traded, shared, or sent to any third party.
  • No Human Inspection: The scanning process is completely automated. No person, including the Developer, can access or read your API responses.

5 Local Storage Details

The extension stores a minimal set of values on your machine via the sandboxed chrome.storage.local API. These elements never leave your local browser:

  • Format Settings: Remembers your preferred document export options.
  • Filter Checkboxes: Persists your chosen file extension preferences.
  • Local Download Log: Keeps a running log of your last 50 compiled files (timestamps, course name, download count) for your own reference. This registry can be reviewed and completely wiped at any time on the Options page.

6 Technical Security & Isolation

To defend your browser session, Classroom Pro Downloader incorporates robust engineering controls:

  • In-Transit Security: All data fetched from Google's API services is secured using encrypted HTTPS (TLS 1.3) protocols.
  • In-Memory Transiency: Scanned materials, file binaries, and metadata are held purely in transient system RAM. As soon as the ZIP file compiler finishes saving the folder structure to your Downloads directory, the memory is instantly freed by the browser.
  • Strict Content Security Policy (CSP): The code operates in compliance with Manifest V3 parameters. There are no remote scripts, dynamically loaded code, or external stylesheets. Everything is packaged locally to prevent security vulnerabilities.

7 The Data Flow Pipeline

This transparent loop details how information moves during a download event:

  1. Client Request: You activate the extension. It uses Chrome's built-in identity API to local-fetch a secure sign-in token.
  2. API Fetch: The extension lists your courses via direct HTTPS calls to Google Classroom API.
  3. Local Download: Attachment links are fetched locally via secure session credentials directly in the background worker.
  4. ZIP Compilation: The local, open-source JSZip utility structures your files into folders in system memory.
  5. Local Write: The browser saves the finished ZIP archive directly to your Downloads folder. The session data is then instantly cleared.

8 Third-Party Services

The code interacts solely with standard, official systems. No third-party trackers are integrated:

  • Google APIs: Direct secure endpoint calls to classroom.googleapis.com.
  • Local JSZip Library: Open-source, compiled internally in your local extension sandbox with no network connectivity.

9 Children's Privacy Compliance

We are fully compliant with Children's Online Privacy Protection (COPPA) standards. Classroom Pro Downloader collects zero personal information, is completely safe for educators and students of all ages, and operates under student-managed Google Classroom security profiles.

10 Retention, Deletion, & Revocation

Because we do not store your data, there are no databases to clear. You can purge the local configuration and remove credentials immediately:

  • Reset Extension: Clear your local download history and checkboxes from the extension options widget.
  • Revoke Tokens: Disconnect the extension instantly via your Google settings.
  • Uninstall: Removing the extension from your browser immediately deletes 100% of its stored parameters.
