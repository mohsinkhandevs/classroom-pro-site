Privacy Policy

Effective Date: May 15, 2026  ·  Product: Classroom Pro Downloader (Chrome & Edge Extension)  ·  Developer: Mohsin Khan

🚫

No Data Collected

We collect absolutely nothing. No accounts, no analytics, no telemetry.

🖥️

100% Local Processing

Everything happens inside your browser. No data ever touches our servers.

This Privacy Policy describes how Classroom Pro Downloader handles your data. By installing or using the extension, you agree to the practices described below. This policy complies with the Google API Services User Data Policy, including its Limited Use requirements.

1 Zero Data Collection

Classroom Pro Downloader does NOT collect, store, transmit, log, or share any of your personal information, Google account data, file contents, or usage statistics with any external server — including ours.

The extension has no backend, no database, and no remote server of any kind. It operates entirely as a client-side browser extension that communicates only with Google's official API infrastructure on your behalf.

  • No user accounts are created or required on our end.
  • No personal identifiers (name, email, Google ID) are ever recorded.
  • No analytics SDK, crash-reporting library, or tracking pixel is included.
  • No file content you download is seen, read, copied, or cached by the Developer.

2 Google API Scopes & Why We Need Them

To function, the extension uses Google's official OAuth 2.0 protocol to access the following scopes. Each scope is the minimum required for that specific feature — we request nothing extra.

Classroom Scopes (Read-Only)

Scope Access Why it's needed
classroom.courses.readonly Read-Only Fetches the list of your active enrolled courses so you can choose which ones to download from.
classroom.coursework.me.readonly Read-Only Reads assignments and coursework items posted to you, including their attached files.
classroom.coursework.students.readonly Read-Only Reads student-visible coursework materials attached to class assignments.
classroom.courseworkmaterials.readonly Read-Only Accesses materials posted directly to a course stream (not part of an assignment).
classroom.announcements.readonly Read-Only Reads course announcements that may contain file attachments you wish to download.

Google Drive Scopes

Scope Access Why it's needed
drive.file Scoped Access Accesses only the specific files you explicitly select via the Google Picker when downloading. Used to (1) read file metadata and download binary content, and (2) convert uploaded Microsoft Office files (e.g. .docx, .pptx) to PDF via a temporary Google Workspace copy that is immediately and permanently deleted. This scope cannot access your entire Drive — only files you actively select.
Note on drive.file: This scope only grants the extension access to files you explicitly select via Google's file picker — not your entire Drive. The extension cannot browse, read, modify, or delete any files you haven't specifically chosen.

You can review and revoke the extension's OAuth access at any time by visiting myaccount.google.com/permissions.

4 Local Storage

The only data stored by the extension is saved locally on your device using the browser's chrome.storage.local API. This data never leaves your device and is never sent anywhere.

  • Export Format Preference: Your last chosen export format (PDF, Word, or original) is saved so you don't have to re-select it each time.
  • File-Type Filter Preferences: Your toggled file-type checkboxes (e.g. "include PDFs", "exclude images") are persisted locally.
  • Download History Log: A local log of your past 50 download sessions (course name, file count, date) is maintained for your own reference. This can be viewed and cleared at any time from the extension's Options page.

You can clear all stored preferences and history at any time by visiting the extension's Options page (click the ⚙️ gear icon inside the extension widget) or by uninstalling the extension. Uninstalling the extension permanently removes all locally stored data.

5 Data Security & Protection Mechanisms

We take the security and protection of your sensitive data seriously. In compliance with the Google API Services User Data Policy, we maintain a secure operating environment and take reasonable and appropriate steps to protect all data accessed via Google API Services against unauthorized or unlawful access, use, destruction, loss, alteration, or disclosure.

Although Classroom Pro Downloader does not collect, store, or transmit any data to external servers, we implement the following industry-standard security measures to protect your data while it is being processed within your browser:

🔐 Encryption in Transit

All communication between your browser and Google's API services is encrypted using TLS 1.2+ (HTTPS). This includes:

  • OAuth 2.0 authentication requests to accounts.google.com
  • Classroom API calls to classroom.googleapis.com
  • Drive API calls to www.googleapis.com for file metadata and content downloads

No unencrypted HTTP connections are ever used. All API endpoints are accessed exclusively over HTTPS.

🔑 OAuth Token Security

Authentication is handled through Google's official OAuth 2.0 protocol via the browser's built-in chrome.identity API. The following safeguards are in place:

  • No tokens are stored persistently: OAuth access tokens are held in browser session memory only during active use and are managed entirely by the browser's identity system.
  • No tokens are transmitted externally: Access tokens are never sent to any server, endpoint, or service other than Google's official API infrastructure.
  • Token scope limitation: Tokens are scoped to the minimum permissions required (read-only Classroom access and scoped Drive access limited to user-selected files via Google Picker). The extension cannot access your entire Drive or modify data you have not explicitly selected.
  • User-revocable: You can revoke the extension's access at any time by visiting myaccount.google.com/permissions.

🧠 In-Memory Processing Only

All sensitive data processing occurs entirely within your browser's volatile memory (RAM). Specifically:

  • File content downloaded from Google Drive is held in memory only during the ZIP packaging process.
  • Course metadata (course names, assignment titles, file names) is used only for organizing the ZIP folder structure and is not persisted after the download completes.
  • Once the ZIP file is generated and saved to your local Downloads folder, all in-memory data (file binaries, metadata, and tokens) is released by the browser's garbage collector.
  • No file content, course data, or user information is written to disk by the extension, except for the final ZIP file saved to your Downloads folder via the browser's native download mechanism.

🛡️ No External Server / No Backend

The extension operates a zero-server architecture. There is no Developer-controlled backend, database, cloud function, or intermediary proxy. This eliminates the following categories of risk entirely:

  • No server-side data breaches — there is no server to breach.
  • No data-at-rest vulnerabilities — sensitive data is never stored at rest on any external system.
  • No third-party data sharing — your data never passes through any system other than your browser and Google's official API infrastructure.

🔒 Content Security Policy (CSP)

The extension operates under Chrome's Manifest V3 security model, which enforces strict Content Security Policy rules:

  • No remote code execution — all JavaScript is bundled locally within the extension package.
  • No inline script injection — the extension's service worker and content scripts are statically declared in the manifest.
  • No third-party script loading — the only external network requests are to Google's authenticated API endpoints.

📋 Access Controls

Access to user data within the extension is limited by design:

  • The extension can only be activated on classroom.google.com pages (as declared in the manifest's content script matches).
  • No developer or third party has access to your Google data — the extension is fully automated and client-side.
  • The extension requests only read-only permissions for Classroom and Drive data. It cannot create, modify, or delete your Classroom assignments, courses, or Drive files (except for temporary copies created during Office-to-PDF conversion, which are immediately deleted).
Summary: Your data is protected by encryption in transit (TLS/HTTPS), secure token handling via the browser's native identity system, in-memory-only processing, and a zero-server architecture that eliminates remote data storage risks entirely.

6 How Your Data Flows

Below is the complete data flow when you use the extension. No step involves any Developer-owned server.

  1. Authentication: You click the extension icon. Your browser authenticates with Google via OAuth 2.0 using Chrome's built-in Identity API. An access token is issued by Google directly to your browser session.
  2. File Discovery: The extension calls the Google Classroom API to list your courses, coursework, materials, and announcements. All results stay in your browser's memory only.
  3. File Download: For each file you select, the extension calls the Google Drive API to download the file binary directly into your browser's memory.
  4. ZIP Packaging: The open-source JSZip library packages all downloaded files into a .zip archive entirely in-memory within your browser tab.
  5. Local Download: The ZIP file is saved to your computer's local Downloads folder via the browser's native download mechanism. No copy of any file is retained in the extension after this step.
  6. (Optional) PDF Conversion: If you convert an Office file to PDF, a temporary Google Workspace copy is created in your own Drive, exported to PDF, and the temporary copy is immediately deleted — all via direct Google API calls from your browser.
At no point in this flow does any data pass through the Developer's servers. The chain is always: Your Browser → Google APIs → Your Device.

7 Third-Party Services

The extension interacts with only the following external services. In each case, your browser communicates directly with the service — no Developer intermediary is involved.

  • Google Classroom API (classroom.googleapis.com) — to list your courses and materials.
  • Google Drive API (www.googleapis.com/drive) — to fetch file metadata and download content.
  • Google OAuth 2.0 (accounts.google.com) — for secure, standard authentication.
  • JSZip (bundled locally, version included in the extension package) — an open-source library used for client-side ZIP creation. It makes no network requests.

We do not integrate with any advertising networks, analytics platforms (e.g. Google Analytics, Mixpanel), crash-reporting services (e.g. Sentry), or any monetization trackers.

9 Data Retention & Deletion

Classroom Pro Downloader retains data only as follows:

  • Google API Data (course info, file metadata, file content): Retained in browser memory only during active use. All data is released from memory immediately after the ZIP download completes or the browser tab is closed. No Google API data is ever persisted to disk or stored beyond the active session.
  • OAuth Access Tokens: Managed by the browser's built-in identity system (chrome.identity). Tokens are session-scoped and are automatically invalidated when the browser session ends. You can also manually revoke tokens at myaccount.google.com/permissions.
  • Local Preferences & History: User preferences (export format, file-type filters) and download history (up to 50 entries) are stored locally via chrome.storage.local. These contain no sensitive data — only setting values, course names, file counts, and timestamps.

How to Delete Your Data

You have full control over all data associated with the extension:

  • Clear local preferences and history: Open the extension's Options page (⚙️ gear icon) and use the "Reset" or "Clear History" buttons.
  • Revoke Google account access: Visit myaccount.google.com/permissions and remove "Classroom Pro Downloader" from your authorized applications.
  • Complete data removal: Uninstalling the extension automatically and permanently removes all locally stored data (preferences, history). Combined with revoking Google access above, this constitutes a complete deletion of all data associated with the extension.
Data Deletion Requests: If you need assistance with data deletion or have any concerns, contact us at mohsinkhan.devs@gmail.com. We will respond within 5 business days.
